# Инструкция по развертыванию Schedule Generator

## Как запустить скомпилированное приложение на другом компьютере

### Вариант 1: Полная установка (Рекомендуется)

1. **Скопируйте следующие файлы и папки на целевой компьютер:**
   ```
   ScheduleGenerator.exe    - Основное приложение
   xlsx_initial/           - Каталог с исходными файлами Excel
   gear_xls/              - Каталог с модулями веб-приложения
   visualiser/            - Каталог с модулями визуализации
   ```

2. **Структура каталогов должна выглядеть так:**
   ```
   [Папка приложения]/
   ├── ScheduleGenerator.exe
   ├── xlsx_initial/
   │   ├── schedule_planning.xlsx
   │   └── newpref.xlsx (создается автоматически)
   ├── gear_xls/
   │   ├── main.py
   │   ├── server_routes.py
   │   ├── Modul1.bas
   │   ├── convert_to_xlsm.py
   │   └── excel_exports/ (создается автоматически)
   └── visualiser/
       ├── example_usage_enhanced.py
       └── другие файлы визуализации
   ```

### Вариант 2: Минимальная установка

Если нужны только базовые функции без веб-приложения и визуализации:

1. **Скопируйте:**
   ```
   ScheduleGenerator.exe
   xlsx_initial/
   gear_xls/Modul1.bas
   gear_xls/convert_to_xlsm.py
   ```

### Системные требования

- **ОС:** Windows 10/11 (64-bit)
- **Microsoft Excel** - для работы с макросами
- **Права администратора** - могут потребоваться при первом запуске

### Возможные проблемы и решения

1. **"Приложение не запускается"**
   - Убедитесь, что все каталоги скопированы в ту же папку, что и .exe файл
   - Проверьте наличие Microsoft Visual C++ Redistributable

2. **"Ошибка при работе с Excel"**
   - Убедитесь, что Microsoft Excel установлен на компьютере
   - Проверьте настройки безопасности Excel для выполнения макросов

3. **"Не найден каталог..."**
   - Проверьте, что все необходимые каталоги находятся в той же папке, что и .exe файл

### Первый запуск

1. Запустите `ScheduleGenerator.exe`
2. Приложение автоматически определит свой рабочий каталог
3. Все функции должны работать так же, как в исходной Python версии

### Примечания

- Приложение создает временные файлы в своем рабочем каталоге
- Для полной функциональности требуется подключение к интернету (для flask-сервера)
- Рекомендуется запускать от имени администратора при первом использовании
