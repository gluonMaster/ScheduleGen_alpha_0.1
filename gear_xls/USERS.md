# Управление пользователями — SchedGen веб-редактор

Все настройки пользователей хранятся в файле `gear_xls/config/users.json`.
Пароли хранятся в виде bcrypt-хешей — открытый текст нигде не сохраняется.

---

## Роли

| Роль | Логин | Что может делать |
|------|-------|-----------------|
| `admin` | alla | Всё: редактировать расписание, публиковать, сохранять в Excel, принудительно снимать блокировку |
| `editor` | valentina | Добавлять/редактировать/удалять **индивидуальные и нахильфе занятия**; просматривать аудитории |
| `viewer` | olesya | Только просмотр расписания и отчёта по аудиториям |

---

## Первоначальная настройка (до первого запуска)

После установки все три пользователя имеют пустые (нерабочие) пароли.
Нужно задать пароли **один раз** из папки проекта:

```
cd C:\путь\к\SchedGen_PreRelease

python gear_xls/scripts/set_password.py alla      <пароль_аллы>
python gear_xls/scripts/set_password.py valentina  <пароль_валентины>
python gear_xls/scripts/set_password.py olesya     <пароль_олеси>
```

После каждой команды в консоли появится: `Password updated for <логин>`

---

## Изменить пароль пользователя

Та же команда — просто запустить повторно с новым паролем:

```
python gear_xls/scripts/set_password.py valentina  <новый_пароль>
```

Перезапуск сервера **не требуется** — новый пароль начинает действовать немедленно.

---

## Если пароль забыт

Просто задать новый пароль той же командой — старый пароль знать не нужно:

```
python gear_xls/scripts/set_password.py alla  <новый_пароль>
```

---

## Изменить отображаемое имя пользователя

Открыть файл `gear_xls/config/users.json` в любом текстовом редакторе
и изменить значение поля `display_name`:

```json
{
  "login": "alla",
  "display_name": "Алла Иванова",
  "role": "admin",
  "password_hash": "..."
}
```

Сохранить файл. Перезапуск сервера **не требуется**.

---

## Изменить роль пользователя

Открыть `gear_xls/config/users.json` и изменить поле `role`.
Допустимые значения: `admin`, `editor`, `viewer`.

```json
{
  "login": "valentina",
  "display_name": "Валентина",
  "role": "admin",
  "password_hash": "..."
}
```

Сохранить файл. Перезапуск сервера **не требуется**.

---

## Добавить нового пользователя

**Шаг 1.** Открыть `gear_xls/config/users.json` и добавить новую запись в массив `users`:

```json
{
  "users": [
    { "login": "alla", ... },
    { "login": "valentina", ... },
    { "login": "olesya", ... },
    {
      "login": "marina",
      "display_name": "Марина",
      "role": "viewer",
      "password_hash": "$2b$12$PLACEHOLDER"
    }
  ]
}
```

**Шаг 2.** Задать пароль через скрипт:

```
python gear_xls/scripts/set_password.py marina  <пароль>
```

Перезапуск сервера **не требуется**.

---

## Удалить пользователя

Открыть `gear_xls/config/users.json` и удалить соответствующую запись из массива.
Сохранить файл. Если удалённый пользователь в этот момент был залогинен,
его сессия истечёт через 8 часов (или он может просто выйти через кнопку «Выйти»).

---

## Технические детали

- Конфиг: `gear_xls/config/users.json`
- Скрипт смены пароля: `gear_xls/scripts/set_password.py`
- Алгоритм хеширования: bcrypt, 12 раундов
- Время жизни сессии: 8 часов
- Смена пароля и роли вступает в силу при **следующем входе** (текущая сессия не прерывается)
